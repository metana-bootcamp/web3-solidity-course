Quick Recap: What is a Group?

A set with a binary operator that has four properties:

  1. Closure: a □ b is always in the set
  2. Associativity: (a □ b) □ c = a □ (b □ c)
  3. Identity: there exists e such that a □ e = a
  4. Inverse: for every a, there exists a' such that a □ a' = e
If the operator is also commutative, the group is abelian.
Quick refresher from last workshop. Four properties define a group. If the operator is commutative too, we call it abelian. All groups we use in ZK happen to be abelian, but commutativity is a bonus, not a requirement for being a group.

Example 1: The Trivial Group

# The set {0} with addition
# Closure: 0 + 0 = 0 ✓ (stays in set)
# Associative: (0+0)+0 = 0+(0+0) ✓ 
# Identity: 0 (adding 0 gives 0) ✓
# Inverse: 0's inverse is 0 ✓

# This is the smallest possible group
# A group CANNOT be empty — it must have at least the identity
Similarly, {1} under multiplication is a group.
These are trivial but valid — the smallest groups possible.
The trivial group exists just to establish that groups can be very small. The important lesson: a group cannot be empty because it must contain at least the identity element. This seems obvious but matters when we discuss the point at infinity in elliptic curves — it's the identity element that ensures the curve points form a group.

Example 2: Real Numbers under ×

Are real numbers a group under multiplication?

  • ✓ Closed: product of reals is a real
  • ✓ Associative: (a×b)×c = a×(b×c)
  • ✓ Identity: 1
  • ✗ Inverse: the inverse of a is 1/a, but 1/0 is undefined!
Zero has no multiplicative inverse.
But ℝ\{0} (reals excluding zero) under × IS a group!

Domain sensitivity: the exact same operation can form a group or not, depending on the set.

This is a crucial lesson about domain sensitivity. The same operation — multiplication — can fail to form a group if you include zero. Remove zero and it works. We saw this with finite fields too: F_p under addition is a group with all elements, but under multiplication we must exclude zero. This sensitivity will come up again with elliptic curves where we add the point at infinity to make the group work.

This dual group structure is what makes finite fields so special and useful for cryptography. A field is essentially a set with TWO group structures.

Example: Matrices as Groups

✓ n×m matrices under +

  • Closed: sum is same dimensions
  • Associative: matrix + is associative
  • Identity: zero matrix
  • Inverse: negate each element

✓ n×n with det≠0 under ×

  • Closed: det(AB) = det(A)×det(B)
  • Associative: matrix × is
  • Identity: identity matrix
  • Inverse: matrix inverse exists
Note: n×n matrices with det=0 under × are NOT even a Monoid!
The identity matrix has det=1, not det=0, so it's excluded from the set.
Matrices give us a great example of a non-commutative group: square matrices with non-zero determinant under multiplication. This is a Group but NOT abelian because matrix multiplication doesn't commute. The determinant being non-zero ensures every matrix has an inverse. det(AB) = det(A) times det(B) ensures closure — the product of non-zero determinant matrices has non-zero determinant.

Example: Polynomials under Addition

from galois import GF, Poly

GF7 = GF(7)

# Polynomials of degree ≤ 3 over GF(7) form a group under addition
f = Poly([3, 2, 5, 1], field=GF7)  # 3x³ + 2x² + 5x + 1
g = Poly([4, 5, 2, 6], field=GF7)  # 4x³ + 5x² + 2x + 6

# Closure: sum is still degree ≤ 3
print(f"f + g = {f + g}")  # coefficients reduced mod 7

# Identity: zero polynomial
zero = Poly([0], field=GF7)
print(f"f + 0 = {f + zero}")  # same as f

# Inverse: negate coefficients (mod 7)
neg_f = -f
print(f"f + (-f) = {f + neg_f}")  # zero polynomial
Polynomials under × are NOT a group — degree increases, breaking closure for bounded degree.
Polynomials over a finite field form a group under addition. This is directly relevant to ZK because proof systems like PLONK and Groth16 work with polynomials over finite fields. Note that polynomial multiplication doesn't form a group because the degree increases — if we bound the degree, multiplication can take us out of the set.

Addition Modulo a Prime

p = 7

# Build the complete group table
print("Addition mod 7:")
print("   | " + " ".join(f"{b}" for b in range(p)))
print("---+" + "-" * (2 * p))
for a in range(p):
    row = " ".join(f"{(a+b) % p}" for b in range(p))
    print(f" {a} | {row}")

# Verify group properties:
# Identity: 0 (row/column 0 gives same element)
# Inverse of 5: 2 (because 5+2 = 7 ≡ 0)
for a in range(p):
    inv = (p - a) % p
    assert (a + inv) % p == 0
    print(f"  Inverse of {a} is {inv}")
Addition modulo a prime is the simplest and most important group for ZK. The output table shows every cell contains a value in {0,...,6}, proving closure. The identity is 0 and each element's inverse is what you need to add to reach 0. This is exactly the additive group of the finite field F_7 that we studied in Workshop 2.

Multiplication Modulo a Prime (Excluding 0)

p = 7

# The set {1, 2, 3, 4, 5, 6} under × mod 7
print("Multiplication mod 7 (excluding 0):")
print("   | " + " ".join(f"{b}" for b in range(1, p)))
print("---+" + "-" * (2 * (p-1)))
for a in range(1, p):
    row = " ".join(f"{(a*b) % p}" for b in range(1, p))
    print(f" {a} | {row}")

# Identity: 1
# Inverse of a: pow(a, -1, p)
for a in range(1, p):
    inv = pow(a, -1, p)
    assert (a * inv) % p == 1
    print(f"  {a} × {inv} ≡ 1 (mod {p})")
The multiplicative group of a finite field excludes 0. This is a group because every non-zero element has a multiplicative inverse when p is prime. This group has order p minus 1 — so for p equals 7, the group has 6 elements. This multiplicative group has a special property we'll explore next: it's cyclic.

Finite Groups & Order

A finite group has a finite number of elements.
The order of a group is the number of elements in it.
  • 𝔽₇ under addition: order = 7
  • 𝔽₇\{0} under multiplication: order = 6
  • All integers under addition: infinite order
In ZK proofs, we ONLY use finite groups.
The order of the group determines the security level.
In ZK proofs, we exclusively work with finite groups. The order — the number of elements — is crucial. For BN128, the elliptic curve group has about 2^254 elements. This large order is what makes the discrete logarithm problem hard and the system secure. If the group were too small, an attacker could brute force the mapping.

Cyclic Groups

A cyclic group is a group where every element can be generated by repeatedly applying the binary operator to a single generator element. 
# Addition mod 7 is cyclic with generator 1:
p = 7
g = 1
print("Generating all elements from g=1:")
element = 0
for i in range(p):
    element = (g * i) % p  # i additions of g
    print(f"  {i} × g = {element}")

# Output: 0, 1, 2, 3, 4, 5, 6 — every element!

Starting from 1 and repeatedly adding 1 generates: 1, 2, 3, 4, 5, 6, 0 — every element of the group.

A cyclic group can be generated from a single element. For addition mod 7, the generator is 1 — keep adding 1 and you hit every element. This is exactly how elliptic curve cryptography works: you pick a generator point G and multiply it by scalars to get all other points. The group being cyclic means you can reach every point from that single generator.

Multiplicative Cyclic Groups

Multiplication mod 7 (excluding 0) — not every element is a generator!

✗ Generator = 2?

p = 7
g = 2
print("Powers of 2 mod 7:")
for i in range(1, p):
    print(f"  2^{i} = {pow(2,i,p)}")
# 2, 4, 1, 2, 4, 1...
# Only generates {1, 2, 4}
# NOT a generator!

✓ Generator = 3!

p = 7
g = 3
print("Powers of 3 mod 7:")
for i in range(1, p):
    print(f"  3^{i} = {pow(3,i,p)}")
# 3, 2, 6, 4, 5, 1
# Generates {1,2,3,4,5,6}
# IS a generator!
An element that generates the entire multiplicative group is called a primitive root.
For multiplicative groups, not every element works as a generator. With mod 7, the element 2 only generates the subgroup {1, 2, 4} — it cycles back to 1 after only 3 steps. But 3 generates ALL six non-zero elements. Elements that generate the whole group are called primitive roots. In elliptic curve cryptography, the chosen generator point G must generate the entire group.

Finding Primitive Roots

from galois import GF

# galois library can find all primitive roots
for p in [7, 11, 13, 23]:
    field = GF(p)
    roots = field.primitive_elements
    print(f"GF({p}): primitive roots = {list(roots)}")
    print(f"  {len(roots)} out of {p-1} elements are generators")
    print(f"  ratio = {len(roots)/(p-1):.1%}\n")

# Manual verification for p=7
p = 7
for g in range(1, p):
    generated = set()
    val = 1
    for _ in range(p - 1):
        val = (val * g) % p
        generated.add(val)
    is_gen = len(generated) == p - 1
    print(f"  g={g}: generates {sorted(generated)} {'← GENERATOR' if is_gen else ''}")
The galois library makes finding primitive roots easy. For p equals 7, the generators are 3 and 5. Notice the ratio — roughly a third of elements are generators, which is related to Euler's totient function. In practice, for elliptic curve cryptography, the generator is specified as part of the curve parameters. For BN128, the generator G1 is a specific point.

Cyclic ⇒ Abelian

If a group is cyclic, it is automatically abelian (commutative).

Proof sketch: Every element = generator applied some number of times.
Let p = g+g+...+g (m times), q = g+g+...+g (n times)
Then p □ q = g added (m+n) times = q □ p = g added (n+m) times
Since m+n = n+m, commutativity follows.
  • Elliptic curve groups in ZK are cyclic → therefore abelian
  • The converse is NOT true: abelian does not imply cyclic
  • ℝ under addition is abelian but NOT cyclic (no single generator)
This is an important theorem. Cyclic groups are automatically commutative. The proof is elegant: since everything is just the generator added to itself repeatedly, the order doesn't matter — m plus n equals n plus m. This is why elliptic curve point addition is commutative. The converse doesn't hold: real numbers under addition are commutative but no single real number generates all others through repeated addition.

The Identity Element is Unique

Proof by contradiction:
Suppose both e and e' are identity elements.
Then a □ a⁻¹ = e, and also a □ a⁻¹ = e'
So e = a □ a⁻¹ = e'
Therefore e = e'.   ∎
  • A group has exactly one identity element
  • For addition groups: the identity is 0
  • For multiplication groups: the identity is 1
  • For elliptic curves: the identity is the "point at infinity"
This seems obvious but is worth proving because it establishes a pattern of mathematical reasoning. The identity is unique — there can be only one. For elliptic curves, this unique identity is the point at infinity, a special point that's not technically on the curve but is added to the set to satisfy the group requirements.

Example: 2D Points as a Group

# Set: all (x, y) pairs with real coordinates
# Binary operator: element-wise addition

# Closure: (a,b) + (c,d) = (a+c, b+d) — still a point ✓
p1 = (1, 1)
p2 = (2, 3)
p3 = (p1[0]+p2[0], p1[1]+p2[1])
print(f"{p1} + {p2} = {p3}")  # (3, 4)

# Identity: the origin (0, 0)
origin = (0, 0)
print(f"{p1} + {origin} = {p1}")  # unchanged ✓

# Inverse: negate both coordinates
inv_p1 = (-p1[0], -p1[1])
print(f"{p1} + {inv_p1} = {(p1[0]+inv_p1[0], p1[1]+inv_p1[1])}")  # (0, 0) ✓
This is a preview of elliptic curves!
EC points are (x, y) pairs with a different addition rule, but same group structure.
2D points under element-wise addition form an abelian group. This is actually a great mental model for elliptic curves! EC points are also (x, y) pairs, the identity is a special point, and the inverse involves negating the y coordinate. The only difference is the addition rule is more complex — but the group structure is identical.

Example: Powers of a Fixed Base

# Set: {b^i : i ∈ ℤ} under multiplication (b = 2)
# {... 1/8, 1/4, 1/2, 1, 2, 4, 8, 16, ...}

b = 2

# Closure: b^x × b^y = b^(x+y) ✓
print(f"2³ × 2⁴ = 2^(3+4) = {b**3 * b**4} = {b**7}")

# Identity: b^0 = 1
print(f"2⁰ = {b**0}")

# Inverse of b^x is b^(-x)
print(f"Inverse of 2³ is 2⁻³ = {b**-3}")
print(f"2³ × 2⁻³ = {b**3 * b**-3}")  # 1 ✓
This is exactly how scalar multiplication on elliptic curves works!
G, 2G, 3G, ... nG wraps around, just like b¹, b², b³... wraps mod p.
Powers of a fixed base form a group under multiplication. This is directly analogous to elliptic curve scalar multiplication. When we compute 1G, 2G, 3G, and so on, we're doing the same thing in additive notation. The exponents form a group under addition that maps homomorphically to this group under multiplication. This connection is actually a homomorphism, which we'll cover in the next workshop.

Groups in Zero Knowledge Proofs

Understanding groups gives you a vocabulary for ZK:

  • "Inverse" → you immediately know what this means
  • "Identity" → the neutral element
  • "Cyclic" → everything comes from a generator
  • "Order" → how many elements before wrapping
  • "Abelian" → order of operations doesn't matter
Groups help us reason about complex objects without understanding their implementation.
If someone tells you EC points form a group, you already know how they behave!
This is the key takeaway. Group theory gives you a powerful vocabulary. When you encounter elliptic curves, bilinear pairings, or polynomial commitments, the underlying math always reduces to group operations. If you know the structure is a group, you know its properties without needing to understand the sometimes terrifying implementation details.

Key Takeaways

  • Groups are sensitive to their domain — same operation, different sets, different structures
  • Finite groups have a specific order (number of elements)
  • Cyclic groups can be generated from a single element
  • Primitive roots generate the full multiplicative group
  • Cyclic ⇒ Abelian: cyclic groups are always commutative
  • Identity is unique: there's exactly one per group
  • In ZK, we exclusively use finite cyclic groups
These concepts form the bedrock for everything that follows. Finite cyclic groups are the specific mathematical structure used in all ZK proof systems. They give us a bounded, structured space where we can perform computations and verify proofs efficiently. Master these concepts and elliptic curves will feel natural.
